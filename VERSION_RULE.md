# 版本控制规则

## 版本号格式

```
主版本.次版本.补丁版本
  0       1       0
```

## 升级规则

| 变更类型 | 升级幅度 | 例子 |
|---------|---------|------|
| 修复错别字、格式调整 | +0.0.1 | 0.3.0 → 0.3.1 |
| 新增1-2个场景对话 | +0.0.1 | 0.3.1 → 0.3.2 |
| 新增1个AI prompt | +0.0.1 | 0.3.2 → 0.3.3 |
| 重组章节、新增小节 | +0.1.0 | 0.3.x → 0.4.0 |
| 哲学框架重构、多章节重写 | +1.0.0 | 0.x.0 → 1.0.0 |

## 当前阶段

**v0.x 阶段**：大量小步迭代，快速成熟期
- 目标：经历 **100+ 次 0.0.1 升级**
- 每个 0.0.1 升级必须包含：至少一个具体的增量（场景/prompt/表述）
- 不追求大版本，追求高频迭代

## 版本历史原则

1. **每次升级必须有实际增量**——不能为了升级而升级
2. **小步快跑**——0.0.1 也是进步
3. **记录每个 0.0.1**——累积量变到质变
4. **大版本慎重**——只有重大重构才 +1.0.0

## 目标

```
v0.1.0  →  初始版本
v0.2.0  →  第一次架构整理
v0.3.0  →  第二次哲学融合
   ↓
   [100+次 0.0.1 小步迭代]
   ↓
v1.0.0  →  技能成熟，可发布
```
